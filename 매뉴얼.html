FAQ

자주 묻는 질문

데이터는 어디에 저장되나요? ▾

브라우저 LocalStorage에만 저장됩니다. 서버 전송은 일체 없고, 같은 브라우저에서만 보입니다. 백업이 필요하면 설정 → 전체 백업 (JSON)으로 받아두세요. 다른 브라우저에서 불러오기로 복원 가능.

자동 생성된 카피를 그대로 발송해도 되나요? ▾

초안으로는 충분하지만 발송 직전엔 반드시 한 번 검토하세요. 회사 톤·법무 가이드라인·과거 발송 사례와 어긋나지 않는지 확인. 트렌디 톤은 격식 채널 (B2B 안내문 등) 에는 부적합할 수 있어요.

저장 버튼이 어디 있나요? ▾

없습니다 · 자동 저장이에요. 입력하는 즉시 LocalStorage에 저장됩니다. 우측 상단의 💾 자동 저장됨 인디케이터가 입력 시 노란색으로 깜빡인 뒤 초록색 ✓ 로 바뀌면 저장 완료.

매핑이 잘못 잡혔는데 다시 하고 싶어요 ▾

데이터 분석 페이지 우측 상단 ✎ 매핑 수정 버튼을 누르면 매핑 모달이 다시 열립니다. 컬럼 변경 → 분석 시작 → 차트 즉시 재생성. 재업로드 불필요.

GitHub Pages 배포는 어떻게 하나요? ▾

새 GitHub 저장소 만들기 → 이벤트.html / index.html / README.md / CHANGELOG.md 4개 파일을 repo 루트에 push → Settings → Pages → Source를 main / (root) 로. 1~2분 뒤 {user}.github.io/{repo}/ 에서 접속 가능.

UTM 파라미터가 뭐예요? ▾

URL 끝에 ?utm_source=email&utm_medium=email 같은 게 자동으로 붙는 기능입니다. GA·Amplitude 같은 분석 도구에서 "어느 채널에서 클릭이 많이 왔는지" 구분할 때 씁니다. 마케팅 분석에 거의 필수예요.

화면이 너무 복잡해요. 좀 정리할 수 있나요? ▾

산출물 페이지에서 서비스 정보 / 디자인 테마 / URL 영역 각각 우측에 ▾ 접기 버튼이 있어요. 자주 안 쓰는 영역은 접어두면 화면이 깔끔해집니다. 우측 상단의 ⇅ 설정 접기로 한 번에 다 접을 수도 있어요.

PRO TIPS

실무 꿀팁 8가지

🎯

템플릿부터 시작

처음엔 5개 템플릿 중 가장 비슷한 걸 골라서 시작하세요. 혜택·KPI·체크리스트가 자동 채워져서 빈 화면 공포가 없어요.

🔥

톤과 디자인을 짝지어 쓰기

차분한 발송 → 격식 톤 + Classic Mono / Cool Navy
인스타광고 → 트렌디 톤 + Y2K Pop / Brutalist

↻

새로 생성을 여러 번

한 번에 마음에 드는 카피가 안 나와도 괜찮아요. ↻ 버튼을 3~5번 누르다 보면 좋은 게 하나는 걸립니다.

📱

SMS는 단축 URL로

SMS는 글자 수에 민감해요. ▶ 채널별 따로 설정에서 SMS만 bit.ly/abc 같은 단축 URL로 지정하면 90자 안에 깔끔하게 들어갑니다.

📧

이메일은 HTML 코드 그대로

이메일 카드의 상세 ↗ → HTML 코드 복사 → 메일 발송 도구에 그대로 붙여 넣으면 실제 작동하는 이메일이 됩니다.

💾

주기적으로 백업

설정 → 전체 백업 (JSON)으로 일주일에 한 번씩 받아두세요. 브라우저 캐시를 지우면 모든 데이터가 날아갑니다.
