01 · Overview

EP Studio가 무엇인가요?

EP Studio는 이벤트·프로모션 기획부터 운영·분석까지 한 화면에서 끝내는 단일 HTML 도구입니다. 별도 서버나 계정이 없고, 모든 데이터는 브라우저에 안전하게 저장됩니다. 회사 IT 보안 정책에 막힐 일도 없어요.

한 줄 요약

기획안 1개 만들면 → 카피 5종 + 배너 미리보기 + 1페이지 결재 문서가 자동으로 따라옵니다.

누구를 위한 도구인가요?

  • 이벤트·프로모션 기획자 — 매번 빈 PPT 화면에서 시작하지 않아도 되도록
  • 마케팅 운영 담당자 — 푸시·SMS·카톡·이메일 카피를 한 번에 뽑아야 할 때
  • 실무 결재 라인 — 1페이지로 정리된 결재 문서가 필요한 모든 곳

3분 빠른 시작

처음 들어왔다면 이 순서대로만 따라가세요.

1
설정 → 회사 정보 입력
사이드바 설정으로 가서 서비스명·회사명·발신자·결재선·문서 헤더를 입력합니다. 이 5개 필드가 모든 카피·배너·기획서에 자동 반영됩니다. (2분 소요)
2
템플릿에서 시작 또는 새 기획안
사이드바 템플릿에서 가장 가까운 5종 중 하나를 골라 이 템플릿으로 시작 또는, 우상단 + 새 기획안 버튼으로 빈 위저드를 엽니다. 7단계 위저드를 따라가면 됩니다. (1분 소요)
3
산출물 페이지에서 결과물 확인
위저드 완료 즉시 산출물 페이지에서 카피·비주얼·결재 문서가 자동 생성됩니다. 톤 4종을 바꿔보거나 새 변형 버튼으로 카피를 다시 뽑을 수 있습니다.
Tip

처음 사용해 보고 싶다면 설정 → 샘플 데이터 채우기로 3개 기획안과 30일 로그를 한 번에 만들어 볼 수 있어요. 데이터를 다 보고 나면 전체 초기화로 깨끗이 비우면 됩니다.

Step 01

기획안 만들기

기획안은 EP Studio의 모든 산출물의 출발점입니다. 한번 잘 만들어두면, 그 뒤로는 자동입니다.

7단계 위저드

단계입력활용처
1이벤트 유형 (리워드·가입·시즌·제휴사·휴면 5종)카피 톤 자동 추천
2기본 정보·일정 (제목·목적·기간·예산)1페이지 결재 문서
3KPI 목표 (지표·목표값·단위)운영 대시보드 비교 기준
4대상 세그먼트 (복수 선택)카피 변수 {seg}
5혜택 설계 (유형·값·단위·한도·조건)모든 카피의 핵심 변수
6채널·URL (발송 채널 + 랜딩 + UTM)채널별 카피 자동 생성
7검토·완료 (상태 결정)—
KPI 빠른 선택

3단계에서 매출·전환율·참여자·재구매율·ROAS·평균객단가 6개 프리셋을 한 번 클릭으로 채울 수 있어요. 직접 입력도 가능합니다.

언제든 수정 가능

기획안은 만든 뒤에도 기획안 페이지에서 수정 버튼으로 같은 위저드를 다시 열 수 있어요. 입력했던 모든 값이 그대로 채워져 있어서 일부만 바꿔도 됩니다.

상태 4종

기획안은 다음 4가지 상태 중 하나입니다.

  • 초안 아직 미정인 상태. 자동 저장만 됨
  • 예정 시작일 도래 전. 결재 받아둔 상태
  • 진행 중 운영 페이지에서 일별 기록 가능
  • 종료 회고·아카이브
Step 02

산출물 자동 생성

이 단계가 EP Studio의 핵심입니다. 기획안 하나로 마케팅 카피 + 비주얼 미리보기 + 1페이지 결재 문서가 즉시 생성됩니다.

상단 톤 칩을 누르면 같은 기획안이지만 다른 어조의 카피로 즉시 전환됩니다. 채널별로 적합한 톤이 달라요.

SMS 자동 분기

SMS 카피 길이가 90자를 넘으면 자동으로 LMS로 표시됩니다. 변형마다 글자 수가 함께 표시돼요.

UTM 자동 부착

기획안에 랜딩 URL을 넣고 UTM 옵션을 켜두면, 카피의 모든 링크에 다음 파라미터가 자동으로 붙습니다.

  • utm_source = 채널 (push·sms·kakao·email·banner)
  • utm_medium = 매체 (notification·sms·social·email·display)
  • utm_campaign = 캠페인 ID
Step 03

운영 · 일별 기록

기획안을 띄운 뒤, 매일 잠깐만 기록해두면 회고할 때 막막함이 없어요.

일별 로그

운영 페이지에서 + 오늘 입력 버튼을 누르고 4개 필드만 채우면 됩니다.

  • 참여자 — 그날 참여한 사용자 수
  • 결제액 — 참여자가 만든 결제 합계
  • 리워드 — 그날 지급한 혜택의 비용 합계
  • 메모 — 푸시 발송 여부·특이사항 등

ROAS(결제÷리워드)와 일평균 예산 대비 진행률이 입력하는 중에 실시간으로 계산돼요.

이슈 로그

운영 중 발생한 이슈를 기록합니다. 심각도 3단계(높음·중간·낮음)와 함께 날짜·제목·상세를 남기면, 회고 PT 만들 때 그대로 활용됩니다.

중요

로그·이슈 모두 행 클릭으로 수정, 우측 🗑로 삭제 가능합니다. 삭제는 되돌릴 수 없으니 확인하세요.

Step 05

인사이트

운영 로그·업로드 데이터에서 자동으로 패턴을 찾아 다음 기획의 단서를 보여줍니다.

자동 패턴 3종

  • 주말 vs 평일 — 어느 요일에 결제가 몰리는지 자동 비교
  • 최고 ROAS 기획안 — 효율 1위 기획안의 혜택 구조 추출
  • 최근 7일 추세 — 직전 7일 대비 증감률 (±20% 초과 시 경고)
언제부터 보이나요?

운영 로그가 최소 7일치 쌓이면 추세 분석이 시작됩니다. 그 전에는 "충분한 데이터가 아직 없어요"라고 표시돼요.

1페이지 결재 문서

산출물 페이지의 📄 결재 문서 탭은 결재 라인이 받아볼 만한 품질의 A4 1페이지 문서를 자동 생성합니다.

구성 요소:

  1. 표지 — 회사 헤더 · 문서번호 (자동 생성) · 작성일 · 제목 · 부제
  2. 01 개요 — 유형·기간·예산·KPI 목표 (일평균 예산 자동 계산)
  3. 02 대상·혜택 — 세그먼트·혜택 구조·1인 한도·적용 조건
  4. 03 채널 — 발송 채널 칩 + 랜딩 URL + UTM 표시
  5. 04 현재 성과 — 운영 로그가 있으면 자동 추가 (4개 KPI)
  6. 05 기대 효과 — 목적·KPI 기준 검토 의견 문단
  7. 결재선 — 회사 정보의 결재선이 자동으로 도장칸으로

내보내기 방법 3종

  • 📋 HTML 복사 — 이메일·메신저·노션에 그대로 붙여넣기
  • ↓ PNG — 2x 해상도 이미지로 저장
  • 🖨 인쇄·PDF — 브라우저 인쇄 → PDF로 저장 → 결재 시스템 첨부

데이터 백업·복원

EP Studio의 모든 데이터는 브라우저 LocalStorage에 저장됩니다. 별도 서버가 없어서 빠르고 안전한 대신, 다른 컴퓨터로 옮기거나 캐시를 지우면 사라질 수 있어요.

설정 → 데이터 관리에서:

  • ↓ 전체 백업 — 기획안·로그·이슈·회사 정보를 모두 포함한 단일 JSON 파일로 저장
  • ↑ 백업 불러오기 — JSON 파일로 한 번에 복원
  • 샘플 채우기 — 3개 기획안 + 30일 로그로 도구 둘러보기
  • 전체 초기화 — 모든 데이터 삭제 (확인 2번 필요)
권장

중요한 기획안을 만든 직후, 또는 한 달에 한 번 정도는 백업 다운로드를 받아두세요. 브라우저 캐시 정리 한 번에 데이터가 날아갈 수 있습니다.

FAQ

자주 묻는 질문

데이터는 어디에 저장되나요? 서버에 올라가나요?
+

아니요. 모든 데이터는 지금 사용 중인 브라우저의 LocalStorage에만 저장됩니다. 외부 서버로 전송되지 않으므로 회사 보안 정책에서도 안전합니다.

대신 다른 컴퓨터·다른 브라우저로 옮기려면 설정 → 전체 백업으로 JSON 파일을 받아 새 환경에서 불러와야 합니다.

AI가 카피를 생성하나요?
+

v0.4 Pro는 AI 통합을 의도적으로 제거한 버전입니다. 모든 카피는 톤·채널별로 정교하게 설계된 룰 기반 템플릿에서 생성되며, 회사 정보·기획안의 변수가 자동으로 채워집니다.

외부 API 키도 필요 없고, 발송 전 검수 과정에서 어떤 카피가 나올지 예측 가능합니다.

카피가 마음에 안 들면?
+

3가지 방법이 있어요.

  • ↻ 새 변형 버튼 — 같은 톤에서 다른 변형으로 즉시 교체
  • 톤 변경 — 혜택강조→한정성→친근→격식 4개 톤으로 분위기 전환
  • 복사 후 수동 편집 — 호버하면 나타나는 ⌘ 복사로 가져가서 직접 수정
1페이지 기획서를 PDF로 받으려면?
+

산출물 → 📄 결재 문서 탭 → 🖨 인쇄·PDF 버튼 → 브라우저 인쇄 창에서 대상을 "PDF로 저장"으로 변경하면 됩니다. A4 18mm 여백으로 자동 설정되어 있어요.

엑셀 업로드가 안 돼요
+

두 가지를 확인해 보세요.

  1. 필수 컬럼이 있는지 — 날짜·결제액 두 컬럼은 반드시 있어야 합니다. 컬럼명은 한국어든 영문이든 상관없어요.
  2. 첫 행이 헤더인지 — 첫 행이 컬럼 이름이어야 합니다. 빈 행이나 합쳐진 셀이 있으면 인식이 어려울 수 있어요.

업로드가 계속 실패하면 ↓ 양식 받기로 권장 포맷의 CSV를 받아 그 형식대로 정리해 보세요.

여러 사람이 같이 쓸 수 있나요?
+

현재 v0.4 Pro는 개인 도구입니다. 협업이 필요한 경우 두 가지 방법을 권장합니다.

  • JSON 공유 — 한 명이 만들고 전체 백업 JSON을 팀 슬랙·드라이브에 공유. 받는 사람은 그 JSON을 불러와서 사용
  • 1페이지 문서 공유 — 결재 문서를 PNG·PDF로 내보내서 공유 (가장 일반적인 방법)
기획안을 실수로 지웠어요
+

아쉽게도 휴지통은 없습니다. 그래서 삭제 버튼을 누르면 확인 창이 한 번 떠요.

최근에 전체 백업을 받아둔 게 있다면 설정 → 백업 불러오기로 그 시점으로 되돌릴 수 있습니다. 이래서 주기적인 백업이 중요해요.

모바일에서도 쓸 수 있나요?
+

기본적으로 데스크톱·태블릿용 도구입니다. 모바일에서도 보이긴 하지만 위저드·산출물 페이지의 레이아웃이 좁아져서 입력 작업이 불편해요. 일별 로그 빠른 입력이나 산출물 확인 정도라면 모바일에서도 무리 없습니다.

실무 꿀팁 5선

  1. 회사 정보부터 채우세요 — 첫 5분만 투자하면 그 뒤 모든 기획안에서 시간이 절약됩니다.
  2. 템플릿으로 시작하세요 — 빈 위저드보다 훨씬 빠릅니다. 마음에 안 들면 어차피 수정 가능해요.
  3. 매일 30초만 운영 기록 — 종료 후 회고 PT 만들 때 데이터가 손에 있는 것과 없는 것은 천지 차이입니다.
  4. 이슈는 발생 즉시 기록 — 시간이 지나면 무슨 일이었는지 까먹어요. "주말 트래픽 -38%, 원인 푸시 미발송" 한 줄이면 충분합니다.
  5. 결재 받기 전 톤 4개 다 돌려보기 — 같은 기획안이 톤 하나로 분위기가 완전히 바뀝니다. 결재 라인 성향에 맞는 톤을 골라 첨부하세요.
