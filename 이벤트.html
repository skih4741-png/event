      How to Use

      사용 가이드

      EP Studio는 5단계 흐름으로 작동합니다. 처음이라면 좌측 하단 "샘플 데이터 채우기"를 누르고 아래 순서대로 둘러보세요.

      1
      데이터 업로드

      결제 CSV/XLSX를 올리면 컬럼을 자동 인식합니다. 잘못 잡혔으면 ✎ 매핑 수정.

      2
      인사이트 확인

      7개 룰 엔진이 패턴을 자동 탐지합니다. Top Insight + 추천 액션.

      3
      기획안 만들기

      7단계 위저드. 템플릿 선택 시 나머지 자동 채움. 또는 추천 액션 → "기획안으로".

      4
      산출물 생성 ⭐

      기획안 카드 "산출물" 버튼. 푸시 · SMS · 카톡 · 이메일 · 배너 · 1페이지 기획서 자동 생성.

      5
      운영 시작

      매일 일별 성과 입력 · 이슈 기록. 진행률 자동 표시. 종료 후 회고.

      자주 묻는 질문
      데이터는 어디에 저장되나요? ▾

      브라우저 LocalStorage에만 저장됩니다. 서버 전송은 일체 없고, 같은 브라우저에서만 보입니다. 백업이 필요하면 설정 → 전체 백업으로 JSON 파일을 받아두세요.

      어떤 데이터 형식을 받나요? ▾

      CSV / XLSX / XLS. 컬럼 이름은 한글 · 영어 모두 인식. payment_date / amount_krw / user_id / category / partner / event_id / reward_krw 같은 표준 이름을 권장하지만, 다른 이름도 자동 매핑되며 수정도 가능합니다.

      자동 생성된 카피를 그대로 써도 되나요? ▾

      초안으로는 충분하지만, 발송 전 한 번 검토를 권장합니다. 회사 톤 · 법무 가이드라인 · 과거 발송 사례와 어긋나지 않는지 확인하세요. v0.4에서 AI 연동을 통해 회사 톤 학습 옵션을 추가할 예정입니다.

      GitHub Pages 배포는 어떻게 하나요? ▾

      새 repo 생성 → docs/ 폴더에 index.html 넣기 → Settings → Pages → Source를 main / docs로. 약 1분 뒤 {user}.github.io/{repo}/에서 접근 가능. 자세한 가이드는 README.md를 참고.

      PNG로 다운받은 이미지를 어떻게 활용하나요? ▾

      생성된 배너 · 푸시 mockup PNG는 검토용입니다. 디자이너에게 전달해 실제 디자인 시안의 기준점으로 쓰거나, 회의 자료에 첨부할 수 있어요. HTML 코드 복사는 이메일 HTML 카드의 경우 실제 발송용으로 그대로 쓸 수 있습니다.
