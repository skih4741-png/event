# 이벤트

> **EP Studio** · v0.3 (Phase 1)
>
> 이벤트·프로모션 기획 → 운영 → 데이터 분석 → 인사이트 도출 → **산출물 자동 생성** 을 한 흐름으로 연결한 단일 HTML 기획 스튜디오.

무료 · 단일 HTML · 서버 없음 · 데이터는 모두 브라우저 안에서만 처리됩니다.

---

## 📁 파일 구조

```
.
├── 이벤트.html       # 메인 앱 (단일 HTML · 약 248KB / 4,157줄)
├── 매뉴얼.html       # 사용 매뉴얼 (인터랙티브 데모 포함 · 82KB)
├── index.html        # GitHub Pages 진입용 — 이벤트.html로 자동 이동
├── README.md         # 이 파일
└── CHANGELOG.md      # 변경 이력
```

위 5개 파일을 모두 **GitHub 저장소 루트**에 두면 그대로 작동합니다.

---

## 🚀 GitHub Pages 배포 (5분)

1. **새 저장소 생성** — GitHub에 새 repository 만들기 (Public 권장)
2. **5개 파일을 루트에 업로드**
   - `이벤트.html`, `매뉴얼.html`, `index.html`, `README.md`, `CHANGELOG.md`
   - 절대 하위 폴더에 두지 말 것 (서브폴더에 두면 `index.html`이 인식 안 됨)
3. **Settings → Pages**
   - Source: `Deploy from a branch`
   - Branch: `main` / `/ (root)` 선택 → Save
4. **1~2분 후 접속**
   - `https://{사용자명}.github.io/{저장소명}/` → 자동으로 `이벤트.html`로 이동
   - 매뉴얼은 `https://{사용자명}.github.io/{저장소명}/매뉴얼.html`

> **주의**: 파일명에 한글이 포함돼 있어요 (`이벤트.html`, `매뉴얼.html`). GitHub 웹 업로드에서는 정상 작동하지만, 일부 git GUI 클라이언트는 한글 파일명을 깨뜨릴 수 있습니다. 그럴 땐 GitHub 웹사이트의 **Add file → Upload files** 로 직접 업로드하세요.

## 📊 데이터 분석 + 매핑

CSV/XLSX 파일을 던지면:

1. **자동 컬럼 인식** — 날짜·결제액·사용자·카테고리·제휴사·이벤트 7개 항목
2. **신뢰도 점수** — 100점 만점, 70점 미만이면 매핑 모달 자동 표시
3. **수동 매핑** — 컬럼별 드롭다운 + 진단 정보 (타입·고유값·결측%)
4. **재매핑** — 차트 본 뒤 ✎ 매핑 수정으로 컬럼 변경 가능 (재업로드 불필요)
5. **매핑 프로필 저장** — 같은 양식 반복 시 한 번에 적용

### 자동 생성 차트 5종
- 기간별 매출 추이 (이벤트 시점 마커)
- 카테고리별 매출 비중
- 시간대별 결제 패턴
- RFM 5×5 고객 세그먼트 (히트맵)
- 이벤트 vs 비이벤트 비교

## 💡 룰 기반 인사이트 7개

데이터를 자동 분석해서 의미 있는 패턴을 찾아냅니다:
- 이벤트 기간 매출 증가율
- 주말 vs 평일 트래픽 차이
- VIP 객단가 vs 평균
- 시간대별 결제 피크
- 휴면 회귀 패턴
- 카테고리별 ROAS
- 신규 vs 기존 사용자 비율

각 인사이트 카드 하단의 **"기획안으로"** 버튼으로 새 기획안 자동 생성.
